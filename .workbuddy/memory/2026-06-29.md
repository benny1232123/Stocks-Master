### ⑥ 24h 守护进程完成（run_daemon.py + smcore/scheduler/）
- 痛点：streamlit 不能 24h 运行，关浏览器就停；不稳定。
- 方案：独立 daemon 进程，纯标准库实现调度器（零依赖，不装 APScheduler/schedule）。
- smcore/scheduler/engine.py：Scheduler 类支持 daily/weekday/interval(trading_hours_only)，主循环 20秒检查，任务在独立线程执行（异常隔离），SIGINT/SIGTERM 优雅退出。
- smcore/scheduler/jobs.py：三个任务——每日选股(调auto_notify_boll子进程)/行情快照刷新(5分钟)/盘中预警(10分钟)。
- 预警逻辑：读 Daily-Action-List CSV→拉实时价→对比 Boll 止损/止盈→触发推企微。加了"接近止盈"(price>=tp*0.98)。
- 实测：TCL科技 现价5.29 接近止盈5.31 → 触发"🎯 接近止盈"预警。
- 入口：run_daemon.py（--once 调试/--status 查状态），scripts/start-daemon.bat 启动。
- 交易日判断：周一到周五 + 时段 9:25-11:35/12:55-15:05。节假日需 job 内部判断（当前未接交易日历）。
- 注意：pip install 被 sandbox 拦截，用纯标准库实现反而更稳。
- daemon 与 streamlit 通过 stock_data/ 文件解耦，互不依赖。

### ⑦ SCF 云端预警完成（不开机也跑）
- 痛点：daemon 要开机才跑，用户想要"不开机也能盘中预警"。
- 方案：腾讯云函数 SCF 定时触发，免费额度内。
- 架构：本地 daemon 跑选股→生成操作清单→上传 COS；SCF 盘中每10分钟从 COS 读清单→拉新浪行情→对比止损止盈→推企微。
- 关键设计：SCF 不用 akshare（依赖太大），改用新浪 HTTP 行情接口（smcore/data/quote_sina.py，只依赖 requests，SCF 预装）。
- smcore/storage/cos.py：COS 上传/下载/列举，环境变量驱动，未配置降级返回 False（不影响本地）。
- scf_alert.py：SCF 入口 main_handler(event, context)，本地可 python scf_alert.py 测试。
- build_scf_package.py：打包 smcore + scf_alert.py 成 zip（45.8KB，极小）。
- DEPLOY_SCF.md：6步部署说明（开通COS/SCF→建桶→配密钥→建函数→配环境变量→配cron触发器）。
- Cron 表达式：0/10 9-14 ? * MON-FRI *（工作日9-15点每10分钟，SCF默认UTC+8）。
- daemon jobs.py 改造：job_daily_pick 后追加 fusion 生成操作清单 + upload_file 上传 COS。
- 实测：TCL科技 现价5.29 接近止盈5.31 → 触发"🎯 接近止盈"预警。COS 未配置时降级正常。
