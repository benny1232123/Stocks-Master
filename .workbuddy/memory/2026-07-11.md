## 01:30 日报白屏真正根因：Number.isInfinity 不存在

用户截图 Console 显示 **`TypeError: Number.isInfinity is not a function`**，发生在 `Array.map`（DailyExpandableList 遍历行渲染时）。
- 我在盈亏比截断代码中用了 `Number.isInfinity(rr)` —— **这不是标准 JS API**。
- 正确写法是 `isFinite(rr)`（全局函数）或 `Number.isFinite()`。
- 修复：commit `c331759`，已 push。等 Render 重建后 `Ctrl+Shift+R` 刷新。

### 教训
- JS 中**不存在** `Number.isInfinity()`。检查无穷大用：
  - `isFinite(x)` → 返回 true 表示有限数
  - `Number.isFinite(x)` → 同上但不做类型转换
  - 直接比较 `x === Infinity || x === -Infinity`
- 写代码时不要"猜 API 名"，用已验证的。

## 01:08 日报"一片空白"根因修复

用户反馈：点击左侧「日报」tab 后右侧**完全空白**（不是"加载中"文字，是纯白）。

### 根因（最终确认）
**`fullDaily` 状态变量从未在应用启动时初始化**。它只在跑完"选股→融合"流程后才被 `reloadArtifacts()` 填充（原代码 line 628，融合完成回调）。如果用户从未跑过选股、或刷新页面后直接点日报 → `fullDaily === null` → 日报页条件渲染走空状态分支。

但为何是"空白"而不是"加载中..."？推测旧版线上可能存在构建/缓存问题导致连空状态都未渲染；或者用户看到的是浏览器缓存的更早期版本。

### 修复
1. **核心修复**（commit `7c43d85`）：挂载 effect 从 `useEffect(() => { loadDailyDates() }, [])` 改为 `useEffect(() => { loadDailyDates(); reloadArtifacts() }, [])`——应用启动即拉取最新日报数据。
2. **防御增强**（commit `09b07bd`）：日报空状态从单行文字升级为醒目卡片（📋 图标 + 标题 + 说明双行），杜绝任何"白屏观感"。
3. 两次提交均已 push（`07dded0..09b07bd`），Render 重建中。

### 经验
- React 页面"白屏"首查：状态依赖的数据是否在对应生命周期加载了。`useState(null)` + 只在特定回调里 `setState` = 首次进入该页必定为空。
- 空状态不能只靠一行小字——用户可能误解为"坏了"。用图标+标题+说明的卡片格式更安全。

### 经验（延续）
- akshare 拉未来日期必失败且易挂起，回测/选股脚本的 K 线请求区间必须限制到"今天+少量缓冲"，不要请求远未来。
- 回测"0 笔交易"要区分：真 K 线拉取失败（全平）→ 提示；窗口未走完（非平浮亏）→ 标注部分回测。两者前端呈现不同。
