# 2026-07-16 工作日志

## daily_backtest.py 去除多线程预拉（用户要求）
**背景**：上一轮二次优化用了 `ThreadPoolExecutor(max_workers=4)` 并发预拉 K 线来提速，但用户反馈「尽量不要用多线程，容易搞崩接口（akshare 上游）」。
**改动**：
1. `scripts/daily_backtest.py`：删除 `from concurrent.futures import ThreadPoolExecutor, as_completed` 和 `from threading import Lock`；预拉阶段改回串行 `for` 循环。
2. 新增 `PREPULL_INTERVAL` 环境变量（默认 0.3s），每只成功/失败后 `time.sleep(interval)`，给上游接口喘息，避免打崩。
3. 进度输出增加 **ETA**（预计剩余时间），更易判断是否会超时。
4. `.github/workflows/daily-pick.yml`：移除 `PREPULL_WORKERS: "4"`，新增 `PREPULL_INTERVAL: "0.3"`。
**补偿速度的保留措施**（依然有效）：`_skip_completed()`（已完成回测的信号日直接跳过，日常增量只需预拉 1~3 天新信号）+ `LOOKBACK_DAYS` 默认 14（减少候选集）。冷启动全量预估 14天×去重约300只×~4s ≈ 20-25min，日常增量 <5min，均在 55min 超时内。

## CI 提交步骤 exit 128 修复
**问题**：`git add -f stock_data/daily_cache/` 和 `stock_data/cache/` 目录不存在（上游步骤失败或未生成）→ `pathspec did not match any files` fatal error → 整步 exit 128 失败。
**修复**：所有 `git add -f` 改为 `git add -f --ignore-missing`（Git 2.25+ 支持），目录/文件不存在时静默跳过而非 fatal；末尾加 `true` 兜底。空提交会被 `diff --cached --quiet` 拦住不产生脏 commit。
