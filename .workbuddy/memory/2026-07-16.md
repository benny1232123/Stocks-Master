## CI git push rebase 冲突 exit 128 修复
**问题**：`git pull --rebase --autostash origin master` 时产生 **CONFLICT**（两次 CI run 同时修改同一只票的 `k_data/*.csv`）→ rebase 中断 → 进入 detached HEAD → `git push` 也失败 exit 128。
**根因**：数据文件是追加/替换型的，两次 CI 可能对同一文件产出不同内容（如某只票的 K 线在前后两天拉取结果略有差异）→ rebase 无法自动合并。
**修复（三层容错推送策略）**：
1. **第1层首选**：`git pull --rebase --autostash origin master` —— 干净线性合并，无冲突时走此路径。
2. **第2层冲突时**：`git rebase --abort` + `git reset HEAD~` 回退 commit + `checkout master` 脱离 detached HEAD → 重新 stage + commit → `git pull --no-rebase -Xtheirs origin master`（merge 策略保留本地最新数据）。
3. **第3层兜底**：如果 merge 也失败 → `git push --force-with-lease origin master` 强推（安全前提：master 上只有 CI 自动提交）。
